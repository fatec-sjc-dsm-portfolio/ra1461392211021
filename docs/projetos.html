1º Semestre - 404-BIS

Descrição
Este projeto consistiu no desenvolvimento de uma plataforma web para terceirização de serviços de manutenção e resolução de problemas de hardware e software. A solução foi projetada para gerenciar tickets de assistência técnica na FATEC, oferecendo uma interface intuitiva e eficiente. Utilizamos a metodologia Scrum para organização e entregas ágeis.

- Python e Flask para o back-end

- HTML, CSS e JavaScript para o front-end

Contribuições pessoais
- Atuei como desenvolvedor back-end, focado no sistema de controle de tickets
- Desenvolvi funções CRUD para manipulação de dados
- Realizei a integração entre front-end e back-end utilizando Flask

2º Semestre – Diário Oficial Monitor

Descrição
Este projeto foi desenvolvido para atender a um desafio empresarial, criando um sistema que automatiza a leitura do Diário Oficial do Estado. A solução identifica citações relacionadas aos associados da empresa parceira, valida essas informações e envia e-mails personalizados para notificação. O objetivo foi otimizar o monitoramento e a comunicação, reduzindo esforços manuais e aumentando a eficiência.

Tecnologias usadas
- Python para o back-end e bot de leitura do Diário Oficial
- HTML, CSS e JavaScript para o front-end
- Banco de dados relacional para armazenamento de dados
- Ferramentas de e-mail (SMTP) para envio de notificações

Contribuições pessoais
- Realizei a integração entre front-end e back-end para exibição e validação de dados
- Modelei e implementei a estrutura do banco de dados para armazenar citações e associados
- Desenvolvi o back-end, incluindo a lógica de leitura do conteúdo e disparo de e-mails

3º Semestre – Pay.me

Descrição
O Pay.me foi desenvolvido para simplificar a gestão de processos financeiros e comerciais, resolvendo problemas como cadastro manual de títulos e dados de endereço. O sistema automatiza o processamento de parcelas, formata dados de moeda e gerencia diferentes tipos de usuários. Tecnologias usadas

- HTML, CSS e JavaScript para o front-end

Contribuições pessoais

- Desenvolvi rotas REST e autenticação
- Integrei as interfaces, por exemplo, login, cadastro e listagem

4º Semestre – Vitality

Descrição
O projeto Vitality, parte do programa API da Fatec São José dos Campos, foi proposto pela MidAll Tecnologias. Desenvolveu estações meteorológicas de baixo custo para coletar dados como vento, chuva, umidade, temperatura e pressão, transmitidos via IoT (Sigfox, LoRa, GSM) para um servidor central. Os dados são exibidos em dashboards web interativos, com foco educacional para estudantes, instalado na Escola Estadual Elmano Ferreira Veloso e na Fatec.

Tecnologias usadas
- Python para o back-end e APIs de recepção de dados IoT
- ReactJS para o front-end e dashboards interativos
- PostgreSQL para armazenamento de dados
- Sigfox, LoRa e GSM para transmissão de dados IoT
- Docker para configuração de ambientes

Contribuições pessoais
- Desenvolvi o back-end, implementando APIs para cadastro e recepção de dados
- Implementei dashboards com gráficos interativos, garantindo usabilidade e clareza
- Colaborei na documentação de rotas e no tutorial educacional sobre meteorologia

5º Semestre – Cloud-Spark

Descrição
O projeto teve como objetivo desenvolver uma aplicação web e mobile que analisa imagens do sensor WPM (CBERS4A), identificando áreas com cobertura de nuvens através de uma IA treinada para gerar máscaras automaticamente. A aplicação permite a escolha da área e período de interesse e viabiliza o download das imagens processadas.

Tecnologias usadas
- TypeScript, React e React Native para o front-end
- Python e YOLO com Roboflow para o serviço de IA
- Prisma e PostgreSQL para banco de dados
- Docker e AWS para infraestrutura em nuvem

Contribuições pessoais
- Realizei a integração entre o front-end e o back-end
- Estruturei o banco de dados com PostgreSQL e Prisma
- Desenvolvi as telas web e mobile com React e React Native

6º Semestre – DeskData

Descrição
O DeskData é uma solução orientada a dados para atendimentos inteligentes. Automatiza o tratamento de informações não estruturadas, gerando dashboards, análises de similaridade, busca semântica com base em processamento de linguagem natural.

Tecnologias usadas
- Python e Pandas para processamento de dados
- TypeScript com Express e React (Vite) para back-end e front-end
- Prisma ORM com PostgreSQL para persistência
- Tailwind CSS e ApexCharts para visualizações
- Docker e AWS para ambientes em nuvem

Contribuições pessoais
- Atuei na integração entre front-end e back-end usando Express e React
- Modelei e implementei o banco de dados com PostgreSQL e Prisma
- Realizei a extração e tratamento dos dados com Python no serviço de IA
